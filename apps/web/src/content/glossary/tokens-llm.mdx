## Qu'est-ce qu'un token dans un LLM ?

Un **token** est la plus petite unité de texte qu'un modèle de langage traite. Contrairement à un mot, un token peut être un fragment de mot, une ponctuation ou un espace. En pratique, 100 tokens correspondent à environ 75 mots en français.

### Pourquoi les tokens sont-ils importants ?

Les tokens définissent deux paramètres critiques pour tout projet IA :

**Le coût** : les API de LLM facturent séparément les tokens d'entrée (le texte envoyé au modèle) et les tokens de sortie (la réponse générée). Les tokens de sortie coûtent généralement 3 à 5 fois plus cher que les tokens d'entrée. Un audit de consommation de tokens avant mise en production évite les dérapages budgétaires.

**La limite de contexte** : chaque modèle a une fenêtre de contexte maximale (exprimée en tokens). Au-delà, le modèle perd les informations les plus anciennes ou rejette la requête. Cette contrainte conditionne l'architecture des agents qui traitent de longs documents.

### Comment optimiser sa consommation ?

Compresser les prompts système, utiliser du caching quand la même instruction est répétée, et tronquer intelligemment les documents longs sont les trois leviers principaux pour réduire les coûts sans dégrader la qualité des réponses.
