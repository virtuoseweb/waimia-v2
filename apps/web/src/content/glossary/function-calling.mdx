## Le function calling : de la génération de texte à l'action réelle

Le **function calling** est la technologie qui transforme un LLM d'un générateur de texte en un orchestrateur d'actions. Sans cette capacité, un modèle de langage ne peut que décrire ce qu'il ferait ; avec elle, il peut le faire réellement.

### Comment ça fonctionne

Le développeur définit un catalogue de fonctions disponibles, chacune décrite par son nom, ses paramètres et leur type. Quand l'utilisateur formule une requête, le modèle analyse si une des fonctions disponibles permet d'y répondre. Si oui, il retourne un JSON structuré avec le nom de la fonction et les valeurs des paramètres à utiliser.

L'application reçoit ce JSON, exécute réellement la fonction (appel API, requête SQL, envoi email) et retourne le résultat au modèle, qui peut alors formuler une réponse finale pour l'utilisateur.

### Pourquoi c'est structurant pour les agents

Le function calling est le mécanisme fondamental qui donne aux agents IA leur capacité d'action. Combiné au Model Context Protocol (MCP), il permet de standardiser l'exposition des outils et de les réutiliser entre différents agents et applications.

La robustesse d'un workflow agent dépend directement de la qualité des descriptions de fonctions : un paramètre mal décrit génère des invocations incorrectes que le modèle ne peut pas auto-corriger.
