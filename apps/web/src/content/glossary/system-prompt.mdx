## Le prompt système : l'ADN de votre agent IA

Le **prompt système** (ou *system prompt*) est l'instruction fondatrice de tout agent ou assistant basé sur un LLM. Il est écrit une fois par le développeur ou le concepteur du workflow, et s'applique à chaque interaction sans être visible de l'utilisateur.

### Que mettre dans un prompt système efficace ?

Un prompt système bien construit répond à six questions :

**Qui est-il ?** Rôle et personnalité : « Tu es l'assistant commercial de [Entreprise], spécialisé dans les solutions logicielles pour PME. »

**Que sait-il ?** Contexte opérationnel : catalogue produit, base de connaissances, données d'entreprise pertinentes.

**Comment répond-il ?** Format de sortie attendu : longueur, structure (listes, JSON, Markdown), niveau de langage.

**Que peut-il faire ?** Outils disponibles et quand les utiliser.

**Que ne peut-il pas faire ?** Limites explicites : sujets interdits, données confidentielles à ne pas divulguer, cas à escalader vers un humain.

**Comment gère-t-il l'incertitude ?** Instructions pour les cas ambigus plutôt que de laisser le modèle improviser.

### Sécurité et maintenance

Le prompt système est une cible pour les attaques de *prompt injection* : un utilisateur malveillant peut tenter d'injecter des instructions contradictoires pour contourner les règles définies. Des techniques de détection et de cloisonnement permettent de renforcer la robustesse des déploiements en production.
