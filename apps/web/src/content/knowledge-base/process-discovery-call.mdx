## Pourquoi un cadre strict pour le 1er appel

Le **discovery call** Waimia dure en général **30 à 45 minutes**. Son objectif n'est pas de « convaincre » un prospect par le discours. Il sert à qualifier rapidement si un projet a une chance réaliste de produire de la valeur.

Sans cadre strict, un premier appel dérive presque toujours vers l'une de ces impasses :

- un échange trop général sur « l'IA » ;
- une démonstration commerciale prématurée ;
- un brief flou sans accès au problème réel ;
- une discussion agréable, mais inutile pour décider.

Le cadre Waimia évite cette dispersion. Il permet de comprendre vite le niveau de maturité, l'urgence, la latitude budgétaire et la capacité de décision. Il protège aussi le prospect : si le terrain n'est pas prêt, mieux vaut le dire tout de suite que vendre un chantier mal engagé.

## Les 8 questions de qualification

Les huit questions suivantes ne sont pas un script figé, mais un cadre de lecture. Chacune a une intention précise.

### 1. Quel est le problème opérationnel exact à résoudre ?

Intention :

- faire sortir le prospect du registre « on veut faire de l'IA » ;
- identifier un irritant métier concret ;
- vérifier si le besoin est lié à un processus réel.

### 2. Quelle équipe est la plus impactée aujourd'hui ?

Intention :

- localiser le point de douleur ;
- identifier les futurs utilisateurs ;
- évaluer si le sujet touche une équipe pilote identifiable.

### 3. Comment ce travail est-il exécuté aujourd'hui ?

Intention :

- comprendre le workflow actuel ;
- détecter les zones manuelles, les reprises, les délais ;
- éviter de concevoir sur un fantasme plutôt que sur un process existant.

### 4. Quelles données et quels outils sont déjà disponibles ?

Intention :

- estimer la [maturité IA](/glossaire/ai-readiness) réelle ;
- vérifier s'il existe des sources exploitables ;
- voir si le sujet dépend d'un [data pipeline](/glossaire/data-pipeline) inexistant.

### 5. Comment mesurez-vous aujourd'hui le coût ou la perte liée à ce problème ?

Intention :

- faire émerger une baseline ;
- vérifier si le futur [ROI IA](/glossaire/roi-ia) pourra être démontré ;
- distinguer douleur perçue et coût observable.

### 6. Pourquoi maintenant ?

Intention :

- tester l'urgence ;
- savoir si le sujet est poussé par une contrainte réelle ou par une curiosité de contexte ;
- évaluer la fenêtre décisionnelle.

### 7. Quel budget ou quel cadre d'investissement est envisageable ?

Intention :

- éviter les appels exploratoires sans moyen d'exécution ;
- calibrer la profondeur du dispositif ;
- savoir si l'on parle d'un audit, d'un pilote ou d'un déploiement plus large.

### 8. Qui décide, qui sponsorise, qui utilise ?

Intention :

- clarifier le circuit de décision ;
- repérer le sponsor métier ;
- éviter les projets portés par un enthousiaste isolé sans pouvoir réel.

## Les 4 critères de fit Waimia

Les réponses aux huit questions sont ensuite lues à travers **quatre critères de fit**. Si l'un d'eux manque complètement, le projet doit être recadré ou refusé.

### 1. Un problème concret et coûteux

Waimia intervient mieux quand le besoin est relié à un processus identifiable, répétitif et mesurable. Si le prospect cherche surtout à « tester l'IA » sans irritant précis, le fit est faible.

### 2. Un minimum de matière exploitable

Il faut au moins des données accessibles, un flux stable, ou une équipe capable de documenter le process. Sans cela, même un bon modèle ne compensera pas l'absence de terrain.

### 3. Un sponsor capable de décider

Un projet sans décideur engagé s'épuise dans les boucles d'alignement. Waimia cherche un owner clair, côté métier ou direction, capable d'arbitrer.

### 4. Une volonté de mesurer

Si le prospect ne souhaite pas définir de baseline ni suivre quelques [KPI IA](/glossaire/kpi-ia), le risque est élevé de finir dans l'impression subjective. Le fit devient alors mauvais, même si le sujet paraît séduisant.

## Ce qui se passe après le discovery call

Le premier appel doit déboucher sur une décision simple, pas sur une zone grise.

### Cas 1 : fit validé

Si les quatre critères sont présents, Waimia propose généralement l'étape suivante la plus sobre :

- audit de maturité ;
- cadrage d'un pilote ;
- ou, plus rarement, industrialisation d'un flux déjà testé.

### Cas 2 : fit partiel

Si le besoin est réel mais que les prérequis manquent, la recommandation peut être plus en amont :

- clarifier les données ;
- stabiliser le process ;
- identifier un sponsor ;
- documenter un cas d'usage avant de revenir.

### Cas 3 : no-fit assumé

Si le projet repose sur trop d'hypothèses, si le budget est incompatible avec l'ambition, ou si le décideur est absent, la meilleure réponse est un refus propre. Cela raccourcit le cycle de vente, protège les deux parties et évite de transformer un mauvais départ en mission fragile.

Le discovery call Waimia n'est donc pas un rite commercial. C'est un filtre opérationnel. Son rôle est de savoir vite si l'on peut vraiment aider, et à quel niveau d'engagement il faut commencer.
